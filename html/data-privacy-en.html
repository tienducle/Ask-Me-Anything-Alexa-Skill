Privacy Policy

Overview

On the web page https://link.pinguincloud.de/ama-en [1] you will be asked to enter additional information that are required to use the Alexa-Skill "Online Lexicon", if you want to use your own OpenAPI API key.

Stored data

Use without own API key

id - The ASK-SDK automatically generates a unique user ID. This ID is unique for each skill, that is in use by the user. The ID is only stored as a SHA-512 hash.
messageHistory - Up to 5 messages, that were sent by the user, will be stored and used in the LLM API requests, to allow the user to ask follow-up questions and provide more context to the GPT model.
usagesByMonth - List with twelve entries, which store the count of requests that were made by the user, to enforce the maximum quota of 30 requests per month. The counter of the upcoming month is always resetted to zero.

Use with own API key

apiKey - The API key is stored encrypted using the SHA-512 hash of the user ID and an additional salt. The API key is only decrypted in-memory by the AWS Lambda backend to send requests to the LLM API.

User Identification

Every user, that activates the skill, gets an individual "Skill User ID" assigned by Amazon Alexa. Your user profile is stored under this ID. A conclusion from this "Skill User ID" to your identity or your Amazon username is not possible to my knowledge. If you deactivate and reactivate the skill, a new "Skill User ID" will be assigned to you. No IP addresses or address information is stored in your user profile.

Right to erasure

You can request a complete deletion of your user profile at any time. Just say "Register" to the Alexa skill and send me an email with the instruction to delete your user profile and provide the generated login credentials that you received. In addition, all records that have not been changed in the database for 90 days (e.g. if the user has not used the skill within 90 days) will be automatically deleted.
