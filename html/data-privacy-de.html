Übersicht

Auf der Webseite https://link.pinguincloud.de/ama-de [1] werden Sie gebeten, Daten einzugeben, die zur Benutzung des Alexa-Skills "Online Lexikon" notwendig sind, falls Sie Ihren eigenen LLM API-Schlüssel verwenden wollten.

Gespeicherte Daten

Nutzung ohne eigenen API-Schlüssel

id - Vom ASK-SDK wird automatisch eine eindeutige Benutzer-ID generiert. Diese ist für jeden Skill des Benutzers unterschiedlich und wird nur als SHA-512 Hash gespeichert.
messageHistory - Bis zu 5 letzte Nachrichten, die der Benutzer an den Skill gesendet hat, werden gespeichert. Diese werden für die Kommunikation mit dem GPT Modell verwendet, damit der Benutzer Folgefragen stellen kann und das GPT Modell den Bezug zu den vorherigen Fragen nicht verliert.
usagesByMonth - Liste mit zwölf Einträgen, die die Anzahl der Anfragen pro Monat speichert, um zu erkennen ob das Limit von 30 Anfragen pro Monat erreicht wurde. Der Zähler des kommenden Monats wird dabei immer genullt.

Nutzung mit einem eigenen API-Schlüssel

apiKey - Der API-Schlüssel wird mit dem SHA-512 Hashwert der Benutzer ID und einem zusätzlichem Salt verschlüsselt gespeichert. Der Schlüssel wird nur im Arbeitsspeicher des AWS Lambda Backends entschlüsselt, um Anfragen an die LLM API zu senden.

Benutzeridentifizierung

Jeder Benutzer, der den Skill aktiviert bekommt eine individuelle „Skill User ID“ von Amazon Alexa zugewiesen. Ihr Benutzerprofil wird unter dieser ID abgespeichert. Ein Rückschluss von dieser „Skill User ID“ auf Ihre Identität oder ihren Amazon Benutzernamen ist nach meiner Information nicht möglich. Wenn Sie den Skill deaktivieren und neu aktivieren wird Ihnen eine neue „Skill User ID“ zugewiesen. Es werden keine IP Adressen oder Adressinformationen in Ihrem Benutzerprofil abgespeichert.

Recht auf Datenlöschung

Sie können jederzeit eine vollständige Löschung Ihres Benutzerprofils beantragen. Sagen Sie beim Alexa-Skill einfach "Registrieren" und schicken Sie mir eine Email mit der Anweisung, Ihr Benutzerprofil zu löschen und geben die generierten Anmeldedaten an, die Sie erhalten haben. Darüberhinaus werden alle Datensätze die in der Datenbank 90 Tage lang nicht verändert wurden (z.B. wenn der Benutzer den Skill innerhalb von 90 Tagen nicht benutzt hat) automatisch gelöscht.
